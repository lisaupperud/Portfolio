Dice Game

Adventure Game

Git Repository

Course: Java B

This project involves creating a text-based adventure game set in a house at night, where the player has to fight off an intruder and call the police to win the game. The project builds on concepts learned in the course, including object-oriented programming, where various locations and interactions are modeled using Java classes. Players will navigate between rooms and interact with objects that affect the game’s outcome. The game world consists of a central location and four rooms. This project emphasizes developing interactive, event-driven logic and understanding how to structure and manage a small-scale game.

Database

Git Repository

Course: Database Managment

This project is about building a Java application that manages job roles using a relational database. It includes full CRUD operations for job roles with fields like role ID, title, description, salary, and creation date. The application also connects employees to job roles and supports user login to view role information, using SQL JOINs. Emphasis is placed on clean resource handling, exception management, and testing, offering hands-on experience with Java and database integration.

Website
Git Repository

Course: Frontend and Frameworks

Course: Agile Project Methodology

This project is a Single Page Application built with the Angular framework, developed in collaboration between 5 Java developers and 4 UX designers. The app is designed to support elderly users in staying active. Users begin by selecting one of four difficulty levels, then choose from four training categories. Based on these choices, they are directed to a session library containing curated workout routines tailored to their preferences.

Go Fish

Git Repository

Course: Testdriven Development

This is a Java-based 'Go Fish' card game. The development follows TDD principles, with a strong emphasis on writing unit tests using JUnit 5. The work is conducted in a pair programming setup together with another student. The application features core components such as Card, Deck, Player, and Hand, and includes game logic for dealing cards, finding specific cards, and interacting between players. The project avoids unnecessary static dependencies and aims to follow clean, modular design. The gameplay logic is thoroughly tested with unit tests to ensure robustness and maintainability.
